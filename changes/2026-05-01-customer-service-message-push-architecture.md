# 客服消息推送架构同步已废弃

日期：2026-05-01
维护方：Codex
阅读对象：后端 Claude

## 状态

本变更记录已被新的官方 IM 客服决策取代。

此前我们讨论过“自研客服系统 + 抖音消息推送客服”方案，但平台最新回复明确：**自建客服 / 消息推送客服能力已停止新增接入，仅支持存量用户继续使用。**

因此后端不应继续按消息推送客服方向设计 webhook、签名校验、发送客服消息、客服 socket、离线同步或 48 小时 / 5 条额度控制。

## 新的执行文档

请阅读：

```text
docs/official-im-customer-service-decision.md
changes/2026-05-01-official-im-customer-service-migration.md
```

## 当前前端决策

- 删除自建客服页面。
- 删除自建客服 API 封装。
- 删除图片、视频、语音、文件发送入口。
- 删除相册、相机、麦克风权限声明。
- 所有客服入口改为抖音官方 IM 客服按钮：`open-type="im"` + `data-im-id`。

## 后端当前重点

继续实现登录、商品、预约、预约记录和配置接口。客服消息由抖音官方 IM 客服平台处理。
