# 协作规则

本仓库用于前端 Codex 与后端 Claude 协作沟通。

## 1. 单一事实来源

- 接口字段以 `docs/api-contract.md` 为准。
- 前端状态以 `docs/frontend-status.md` 为准。
- 任务状态以 `tasks/backend-todo.md` 和 `tasks/frontend-todo.md` 为准。

## 2. 修改接口的流程

任何一方需要新增或修改接口字段时：

1. 先修改 `docs/api-contract.md`。
2. 在对应任务文件中记录修改原因。
3. 另一方阅读后再实现。

不要先私自改代码再让另一方猜字段。

## 3. 状态标记

任务状态统一使用：

- `[ ]` 未开始
- `[~]` 进行中
- `[x]` 已完成
- `[!]` 阻塞，需要对方确认

## 4. 禁止提交内容

本仓库不要提交：

- token
- 密钥
- Cookie
- 真实用户手机号
- 真实用户聊天记录
- 真实订单隐私数据
- 服务器账号密码

## 5. 业务边界

当前版本固定边界：

- 不做支付。
- 不做退款售后入口。
- 不做交易订单同步。
- 只做预约/预定记录。
- 不展示门店地址。
- 客服是自建在线客服。

如要改变这些边界，必须先在本仓库文档里明确说明，再改代码。

## 6. 沟通格式建议

新增问题建议写成：

```md
## 问题标题

提出方：前端/后端
日期：YYYY-MM-DD
状态：[ ] 未处理

### 背景

### 需要对方确认

### 建议方案
```

## 7. 阅读后写回规则

从 2026-04-30 起，前后端 AI 读取对方的新回复或新方案后，必须把分析和建议写回本仓库。

要求：

- 不能只在聊天窗口口头总结。
- Codex 读取 Claude 回复后，应在 `changes/` 新建或更新 Codex 回复文件。
- Claude 读取 Codex 回复后，应在 `changes/` 新建或更新 Claude 回复文件。
- 回复文件必须包含：已读文件、总体结论、接受项、分歧项、建议调整、需要用户拍板的问题。
- 如果涉及接口变化，必须同步提醒更新 `docs/api-contract.md`。
- 如果涉及任务变化，必须同步提醒更新 `tasks/backend-todo.md` 或 `tasks/frontend-todo.md`。

建议命名：

```txt
changes/YYYY-MM-DD-codex-response-to-*.md
changes/YYYY-MM-DD-claude-response-to-*.md
```

## 8. 联调前置条件

后端准备联调前，需要提供：

- HTTPS API 域名
- 抖音开放平台 request 合法域名配置确认
- 登录接口可用
- 商品接口可用
- 预约接口可用
- 客服会话接口可用

前端联调时会把：

```ts
USE_MOCK = false
API_BASE_URL = 'https://实际后端域名'
```
